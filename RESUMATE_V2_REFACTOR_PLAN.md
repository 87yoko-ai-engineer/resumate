# ResuMate v2 既存アプリ改修計画

## Standing AI Instructions

新セッションでこのリポジトリを扱うAIは、まずこのファイルを読むこと。

この作業は新規開発ではなく、既存アプリの段階的な大改修である。既存アプリを作り直さず、現在の機能を壊さないように進める。

必ず専用ブランチを切って作業する。指定がなければ `codex/` プレフィックスのブランチを使う。

最初に実装するのは Phase 1 のみ。

Phase 1 では、画面構成を次の見え方に整理する。

1. `① 求人情報を貼り付け`
2. `② 履歴書作成（画像添付・音声入力・直接入力）`

Phase 1 では、AIロジックや画像読み取り機能の本実装は行わず、既存機能を壊さないUI整理を優先する。

Phase 1 の実装が終了し、動作確認ができたら、ブランチを切っているか確認し、`.gitignore` を確認し、セキュリティ面を考慮したうえで、対象差分だけをコミット＆pushする。

Phase 2 以降も同様に、1フェーズずつ実装、確認、コミット、push の順で確実に進める。

## 確定仕様：全体フローとAIヒアリング（最重要・厳守）

このアプリの中心価値は「求人分析」と「応募者の経歴」を**両方踏まえて**、本人の魅力を引き出す質問をすることである。実装・改修時は必ずこの仕様を守る。

### 全体フロー
1. **① 求人募集を貼り付け**（任意・スキップ可）
2. **② AIの分析**（①が入力されたときのみ発動。求人から「求める人材像」等を分析）
3. **履歴書入力**（画像添付／音声入力／フォーマット直接入力／チャット欄からのフォーマット入力）
4. **AIヒアリング開始**（ボタンは1つに統一）
   - ①が入力済み：ボタン名「**求人分析内容と応募者の経歴を踏まえたAIのヒアリングを開始する**」
   - ①がスキップ：ボタン名「**応募者の経歴を踏まえたヒアリングを開始する**」
   - 旧「棚卸しを開始する」ボタンは廃止し、この1つに統一する。

### ※ AIヒアリングの質問内容（厳守）
- 氏名などの自己紹介は、履歴書作成欄（画像・音声・フォーマット入力）で入力済みのため、**質問項目に絶対に含めない**。履歴書から分かることを一から聞き直さない。
- 材料にするのは履歴書の次の項目：
  ① 職歴 ② 免許・資格 ③ 職務経歴 ④ 職務要約 ⑤ 活かせるスキル ⑥ 志望動機 ⑦ 自己PR
- これらから、本人の魅力を引き出せそうな質問を作ってヒアリングし、より魅力的な
  **職務要約／自己PR** の2項目を作成する（**両方とも必ず提案する。職務要約だけで終わらせない**）。
- 使用者が**承認すれば、同項目を上書き**する（proposeImprovement → 承認カード → 上書き）。
- 出力（ブラッシュアップ提案）は職務要約・自己PRの2つに限定。スキル・志望動機などは対象外（材料としては考慮する）。

**求人分析がある場合**：求人からAIが分析した内容と、履歴書の①〜⑦を**照らし合わせて**、魅力を引き出す質問を作りヒアリングし、上記2項目（職務要約・自己PR）をより魅力的に作成する（両方提案・承認で上書き）。

**求人分析がない（スキップ）場合**：履歴書の①〜⑦のみから、魅力を引き出す質問を作りヒアリングし、上記2項目（職務要約・自己PR）をより魅力的に作成する（両方提案・承認で上書き）。

### 将来拡張（画像添付・音声入力）
画像添付・音声入力で取り込んだ情報も、最終的に履歴書データ（ResumeData）を更新する形にし、フォーム・プレビュー・ヒアリングすべてに反映されるようにする。

## Summary

ResuMate v2 は、既存の履歴書作成アプリを作り直さず、現在の機能を活かして「求人情報」と「応募者の経歴」をAIがつなぐキャリアアドバイザー型サービスへ改修する。

新しい画面構成は次の2本柱にする。

1. **求人情報を貼り付け**
   求人票・募集要項・URLを入力し、AIが企業の求める人物像を分析する。求人情報は任意入力とし、未入力でも履歴書作成は進められる。

2. **履歴書作成**
   入力方法をカード形式で選べるようにする。
   - 画像添付: 写真、スクショ、PDFから履歴書情報を読み取る
   - 音声入力: 質問シート形式で話して入力する
   - 直接入力: 既存フォームで手入力・修正する

初回の大改修では、画像/PDF読み取りや音声シートをすべて完成させるより先に、AI助言機能の完成度を上げることを最優先にする。

## Key Changes

- 既存の求人分析、AIチャット、履歴書/職務経歴書フォーム、AIブラッシュアップ、写真撮影、localStorage保存、印刷/PDF出力、APIキー設定は残す。
- 画面導線を `① 求人情報を貼り付け`、`② 履歴書作成` の順に整理する。
- `② 履歴書作成` には `画像添付`、`音声入力`、`直接入力` の3カードを表示する。
- 初回改修では `画像添付` と `音声入力` は将来拡張も見据えた入口を整え、既存の直接入力フォームとAI助言体験を優先して完成させる。
- AIチャットを、求人分析と応募者情報を見比べて不足情報を質問し、魅力的な書き方を提案するAIキャリアアドバイザーに変更する。
- AIが職歴、自己PR、志望動機を書き換える場合は、勝手に反映せず、必ず提案後に本人が承認してから反映する。

## Implementation Plan

### Phase 0: 既存仕様確認

- 実装前に `AGENTS.md` の指示に従い、Next.js 16 の関連ドキュメントを `node_modules/next/dist/docs/` で確認する。
- 既存アプリを削除・作り直しせず、現在の `app/page.tsx`、`components/job-posting-input.tsx`、`components/chat-panel.tsx`、`components/resume-preview.tsx`、`lib/prompts.ts`、`lib/resume-schema.ts` を中心に改修する。
- 既存の写真撮影機能は特徴として残す。

### Phase 1: 画面構成の整理

- 画面上の導線を `① 求人情報を貼り付け`、`② 履歴書作成` に整理する。
- 求人情報エリアは任意入力として表示し、求人分析済みの場合はAI助言に使われることが分かる状態にする。
- 履歴書作成エリアには3つの入力カードを置く。
  - `画像添付`: 写真、スクショ、PDF対応予定
  - `音声入力`: 質問シート形式で入力
  - `直接入力`: 現在の履歴書/職務経歴書フォームを開く
- 既存の履歴書フォーム、職務経歴書フォーム、写真撮影、印刷/PDF出力は壊さず維持する。

### Phase 2: AIキャリアアドバイザー強化

- `lib/prompts.ts` のチャットプロンプトを改修し、AIの役割を明確にする。
- AIは求人分析がある場合、応募者情報と照合して次を行う。
  - 企業が求める人材像に近い経験を見つける
  - 書類上で広げられそうな経験を深掘りする
  - 不足している情報を1問ずつ質問する
  - 数字、規模、役割、成果、工夫、困難の乗り越え方を聞き出す
- 求人分析がない場合も、一般的なキャリア棚卸しとして使えるようにする。
- AIは作り話や過度な誇張をしない。本人が話した事実をもとに言い換える。

### Phase 3: 提案・承認フロー

- AIが自己PR、志望動機、職務要約、スキル欄を改善する場合、元文と改善案を区別して見せる。
- 改善案には短い理由を添える。
- 反映操作は本人の承認後に限定する。
- 既存の `BrushupDialog` は承認UIとして活かし、チャット中のAI提案には専用の承認ダイアログを使う。

### Phase 4: 画像・PDF入力

- 写真、スクショ、PDFをアップロードできる入口を追加する。
- 初期実装では、読み取った内容を即時確定せず、AIが抽出した項目を確認画面に出す方針にする。
- 抽出対象は、氏名、ふりがな、生年月日、住所、電話、メール、学歴、職歴、資格を優先する。
- 読み取り結果は本人が確認・修正してから履歴書データに反映する。
- 個人情報は既存方針どおりサーバーに保存しない。

### Phase 5: 音声入力シート

- 音声入力は自由会話ではなく、質問シート形式にする。
- 氏名、住所、学歴、職歴、資格などを項目ごとに話して入力できるようにする。
- 郵便番号が話された場合は既存の郵便番号補完を活用し、住所候補を出す。
- 音声認識結果は必ず手入力で修正できるようにする。

## Public Interfaces / Data Changes

- `ResumeData` は既存構造を基本維持する。
- AI提案用に `AdvisorSuggestion` を追加する。
  - `targetField`
  - `originalText`
  - `suggestedText`
  - `reason`
- `JobAnalysis` は既存項目を維持しつつ、任意項目として `requiredSkills`、`preferredTraits`、`appealPoints` を追加できる。
- localStorage保存方針は維持し、ログインやDB保存は今回の対象外にする。
- APIキーは既存のBYOK方式を維持する。

## Test Plan

- 求人情報なしで履歴書作成を開始できる。
- 求人情報を貼り付けて分析すると、AI質問が求人内容を反映する。
- 直接入力フォームで既存どおり履歴書・職務経歴書を編集できる。
- AIが質問した内容が履歴書データに反映される。
- AI改善案は自動反映されず、承認後にのみ反映される。
- 写真アップロード・カメラ撮影が既存どおり使える。
- localStorageでリロード後も内容が復元される。
- 印刷/PDF出力のレイアウトが大きく崩れない。
- APIキー未設定時は既存どおり設定ダイアログへ誘導される。
- `npm run lint` と `npm run build` が通ることを確認する。

## Assumptions

- この計画は新規開発ではなく、既存アプリの段階的な改修計画である。
- 初回改修の最優先は、画像/PDF/OCRの完全実装ではなく、AIキャリアアドバイザー機能の完成度向上である。
- 求人情報入力は任意とする。
- AIの書き換えは本人承認後に反映する。
- 個人情報はサーバー保存せず、現在のブラウザ保存方針を維持する。
- 既存の証明写真撮影機能は、実用品質に限界があってもサービスの特徴として残す。
